# Прогноз покупки клиента в течение 90 дней

Проект решает задачу бинарной классификации для интернет-магазина: нужно оценить вероятность того, что клиент совершит покупку в течение следующих 90 дней.

## Данные

Используются таблицы:

- `apparel-purchases.csv` — история покупок клиентов;
- `apparel-messages.csv` — история рекламных рассылок;
- `apparel-target_binary.csv` — бинарный целевой признак;
- `full_campaign_daily_event.csv` — дневная агрегация событий рекламных кампаний;
- `full_campaign_daily_event_channel.csv` — дневная агрегация событий рекламных кампаний по каналам.

В проект включён архив `data/filtered_data.zip`. Ноутбук также поддерживает вариант, когда CSV-файлы распакованы в папку `data/`.

## Цель

Максимизировать метрику `ROC-AUC` для прогноза покупки клиента в течение 90 дней.

```bash

```

3. Запустите все ячейки сверху вниз.

## Особенности решения

- Большая таблица `apparel-messages.csv` читается чанками, чтобы снизить потребление памяти.
- Категории товаров обрабатываются как набор ID, потому что один и тот же ID может встречаться на разных уровнях дерева категорий.
- Для `nunique_*` колонок агрегатов кампаний не используется суммирование по датам, так как это привело бы к некорректному подсчёту уникальных клиентов.
- Для оценки качества используется `ROC-AUC`, так как классы несбалансированы.
- Финальная модель выбирается через `GridSearchCV`.
