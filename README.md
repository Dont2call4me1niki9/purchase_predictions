# Marketing Purchase Prediction

Проект для прогнозирования вероятности покупки клиента интернет-магазина в течение следующих 90 дней.

## Цель

Построить модель бинарной классификации и максимизировать метрику ROC-AUC.

## Данные

Используются файлы:

- `apparel-purchases.csv` — история покупок;
- `apparel-messages.csv` — история рекламных рассылок;
- `apparel-target_binary.csv` — целевой признак;
- `full_campaign_daily_event.csv` — агрегаты событий по кампаниям и дням;
- `full_campaign_daily_event_channel.csv` — агрегаты событий по кампаниям, дням и каналам.

Если `apparel-messages.csv` находится внутри архива `filtered_data-20260603T064953Z-3-001.zip`, ноутбук автоматически извлечёт его.

## Структура проекта

```text
marketing_purchase_prediction.ipynb
README.md
requirements.txt
```

## Основные этапы

1. Загрузка данных.
2. Первичный анализ:
   - размер таблиц;
   - типы данных;
   - пропуски;
   - дубликаты;
   - баланс целевого класса.
3. Генерация признаков:
   - RFM-похожие признаки по покупкам;
   - признаки активности за последние 7/14/30/60/90 дней;
   - признаки по популярным категориям;
   - признаки по взаимодействию с рассылками;
   - конверсионные признаки по событиям рассылок.
4. Обучение моделей:
   - Logistic Regression;
   - Random Forest.
5. Подбор гиперпараметров и финальное тестирование по ROC-AUC.

## Запуск

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Поместите CSV-файлы рядом с ноутбуком.

3. Запустите ноутбук:

```bash
jupyter notebook marketing_purchase_prediction.ipynb
```

## Важные замечания

- Целевой класс несбалансирован, поэтому используется стратифицированное разбиение и `class_weight='balanced'`.
- Колонки `nunique_*` из агрегированных таблиц нельзя суммировать по дням, так как уникальность рассчитана внутри каждого дня.
- Основная метрика проекта — ROC-AUC.
